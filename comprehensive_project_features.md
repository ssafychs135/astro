## 1. 🔵 Blue Fox (S14P11A702)
**"가장 민첩한 AI 성능 최적화 파트너"**

### 🧠 핵심 비즈니스 로직 (How it works)
1. **무중단 데이터 수집**: `Byte Buddy` 기반 Java Agent가 타겟 애플리케이션의 실행 흐름을 가로채어 메서드 실행 시간과 SQL 쿼리를 `Span` 단위로 기록합니다.
2. **콜스택 트리 재구성**: 수집된 수천 개의 `Span`을 부모-자식 관계로 매칭하여 시각화 가능한 `CallStack` 트리 구조로 변환합니다.
3. **자동화된 진단 루프**: GitHub PR이 생성되면 서버가 이를 감지하여 성능 테스트를 트리거하고, 분석된 데이터를 AWS Lambda(AI)로 보내 최적화된 코드를 PR에 직접 코멘트로 남깁니다.

### 🛠 비즈니스 로직별 적용 기술
| 비즈니스 기능 | 적용 기술 및 라이브러리 | 선정 사유 |
| :--- | :--- | :--- |
| **코드 계측 (Instrumentation)** | `Byte Buddy`, `Java Agent` | 소스 수정 없이 런타임에 성능 측정 로직을 주입하기 위함 |
| **데이터 전송** | `gRPC`, `Protobuf` | 수천 개의 Span 데이터를 저지연으로 수집 서버에 전송하기 위함 |
| **대용량 데이터 저장** | `MongoDB` | 스키마가 가변적이고 쓰기량이 많은 트레이스 데이터를 저장하기 위함 |
| **실시간 UI 업데이트** | `SSE (Server-Sent Events)` | 성능 테스트 진행 상황을 실시간으로 프론트엔드에 스트리밍 |
| **AI 진단 및 분석** | `AWS Lambda`, `GPT-4/Solar` | 무거운 AI 연산을 서버리스로 분리하여 메인 서버 자원 보호 |

## 🍷 2. Jupasu (S14P21A505)
**"데이터 기반의 지능형 와인 큐레이션 및 소셜 서비스"**

### 🧠 핵심 비즈니스 로직 (How it works)
1. **벡터 임베딩 검색**: 와인의 맛, 향, 정보를 고차원 벡터로 변환하여 `pgvector`에 저장합니다. 사용자의 자연어 질문을 동일한 벡터 공간으로 투영하여 가장 가까운 와인을 추천합니다.
2. **시각적 와인 매칭**: 와인 라벨 이미지를 분석(OCR)하여 데이터베이스 내의 고유 와인 ID와 매칭함으로써 즉각적인 정보를 제공합니다.
3. **개인화 취향 엔진**: 사용자의 스크랩, 리뷰, 평점 데이터를 분석하여 '나만의 맛 레이더' 리포트를 생성합니다.

### 🛠 비즈니스 로직별 적용 기술
| 비즈니스 기능 | 적용 기술 및 라이브러리 | 선정 사유 |
| :--- | :--- | :--- |
| **벡터 유사도 검색** | `PostgreSQL`, `pgvector` | RDB 환경 내에서 SQL과 벡터 검색을 동시에 처리하기 위함 |
| **이미지/라벨 분석** | `FastAPI`, `OpenCV`, `Tesseract` | 고성능 Python 환경에서 라벨 인식 및 이미지 전처리를 수행 |
| **서버 간 통신** | `Spring WebClient` | 백엔드와 AI 서버 간의 비동기 논블로킹 데이터 교환 |
| **추천 알고리즘** | `Cosine Similarity (<=>)` | 와인 맛 데이터 간의 방향성(취향) 유사도를 가장 잘 반영함 |
| **데이터 캐싱** | `Redis` | 빈번한 와인 상세 정보 조회 및 사용자 취향 데이터의 빠른 접근 |

## 🔍 3. Team9 Project (team9)
**"AI 기반 동영상 요약 및 RAG 전문 상담 플랫폼"**

### 🧠 핵심 비즈니스 로직 (How it works)
1. **Self-Querying RAG**: 사용자의 질문에서 "아이폰 14", "2023년 이후"와 같은 메타데이터를 AI가 스스로 추출하여 검색 범위를 좁힘으로써 답변의 정확도를 극대화합니다.
2. **동영상 컨텍스트 추출**: 유튜브 비디오 ID를 기반으로 자막을 수집하고, 이를 AI가 분석하여 핵심 내용을 구조화된 텍스트로 요약합니다.
3. **지능형 라우팅**: 질문의 성격에 따라 '데이터 검색이 필요한 질문'과 '일반 대화'를 분리하여 응답 속도와 API 비용을 최적화합니다.

### 🛠 비즈니스 로직별 적용 기술
| 비즈니스 기능 | 적용 기술 및 라이브러리 | 선정 사유 |
| :--- | :--- | :--- |
| **RAG 워크플로우** | `LangChain`, `SelfQueryRetriever` | LLM이 스스로 필터링 조건을 생성하고 지식 베이스를 검색하게 함 |
| **벡터 데이터베이스** | `ChromaDB` | 오픈소스 환경에서 빠르고 가볍게 벡터 검색 기능을 구현 |
| **한국어 특화 답변** | `Upstage Solar`, `solar-pro2` | 한국어 문맥 이해도가 가장 높은 모델 중 하나를 선택 |
| **동영상 자막 수집** | `Node.js`, `youtube-transcript-api` | 비동기 I/O가 뛰어난 Node.js 워커를 통해 대량의 자막 수집 처리 |
| **동적 웹 크롤링** | `Selenium`, `WebDriverManager` | JS로 렌더링되는 쇼핑몰의 리뷰 데이터를 안정적으로 수집 |
